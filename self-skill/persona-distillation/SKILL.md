# Persona Distillation · 人格/视角蒸馏术

> 改造自女娲方法论 [alchaincyf/nuwa-skill](https://github.com/alchaincyf/nuwa-skill)（创建者 [花叔](https://x.com/AlchainHust)），
> 适配本体系（Hermes profile / SOUL / delegate_task）。核心方法论（心智模型三重验证、表达 DNA、矛盾保留、诚实边界、Agentic Protocol）全部保留。

## 核心理念

蒸馏不是复制人，是**提炼思维框架**——捕捉 HOW they think，不是 WHAT they said。

一份好的人格 skill 是一套可运行的认知操作系统：
- 用什么**心智模型**看世界？（镜片）
- 用什么**决策启发式**做判断？（直觉规则）
- 怎么**表达**？（DNA）
- **绝对不会**做什么？（反模式）
- 这个 skill **做不到什么**？（诚实边界）

## ⛔ 第一铁律：禁止闭门造车（所有蒸馏不可逾越）

> **任何蒸馏项目，必须用真实联网检索获取一手资料，绝对禁止仅凭训练知识/记忆编造。**
> 这是所有 Hermes 实例在所有蒸馏项目中共同遵守的硬门，不分级别、不分人物/主题、不分自动/手工。

为什么单列成铁律：**实测最常见、最隐蔽的失败**就是——子 agent（或主 agent）明明有联网能力，却不调用，凭训练记忆编出"听起来对但不存在的语录/数据/近况"，甚至谎称"无联网权限"。这种产物看似完整实则有毒，会让人格在对话里反复输出假引语。

**硬性要求（每条都要做到）**：
1. **必须真实发起检索调用**——每个调研维度都要有实际的 web 检索/抓取动作，不能空手凭记忆。
2. **每条事实附真实可访问 URL**——没有 URL 的论断一律标 `[INFERRED-未验证]`，不得当事实交付。
3. **逐字引语必须来自真实抓取的页面**——凭记忆复述的标 `[paraphrase]`，**绝不伪造引号原文**。
4. **派 subagent 调研时，主 agent 必须在 context 写死这条铁律**，并在收回结果时**核对 tool_trace 是否真有检索调用**——没有 = 没做，打回重来或主 agent 自己补检索。
5. **做不到就如实说"检索失败/信息不足"**，标进诚实边界——宁可残缺也不编造（general rule §2.10 显式失败优先于编造）。

**本环境实测可用的检索通道**（无需特殊权限）：
- `curl` 直连维基 REST API（`https://en.wikipedia.org/api/rest_v1/page/summary/<title>`）、SEC EDGAR、官方 IR 页
- `curl "https://html.duckduckgo.com/html/?q=<query>"` —— 拿真实结果列表（标题+URL+摘要）
- `curl "https://r.jina.ai/<URL>"` —— 读任意页面全文为 markdown（有 CAPTCHA 的站会失败，换源）
- `web_search` / `browser` toolset（若已启用）/ `webworms` skill（结构化爬取）

> **验收**：交付前必须能回答"这句话/这个数字是从哪个 URL 来的？"。回查不到来源的内容 = 不合格，回 Phase 2 补检索。

## 两类产物（先分清）

| 维度 | 人物 skill | 主题人格 |
|------|-----------|---------|
| 核心 | 一个真人的思维方式 | 一个角色原型 / 领域方法论 |
| 心智模型来源 | 主要来自一个人 | 综合多人视角 + 领域共识 |
| 表达风格 | 模拟此人语气 | 中性但专业，有职业辨识度 |
| 矛盾处理 | 保留个人内在矛盾 | 呈现流派间分歧 |
| 验证方式 | 对比此人已知立场 | 对比领域内经典案例 |
| 角色扮演 | 用「我」第一人称扮演 | 可第一人称扮演角色原型，无需绑定真名 |

> **路径选择**：用户的需求指向某个具体真人的思考方式 → 人物 skill；指向某个职业/领域的方法论 → 主题人格；不确定 → 推荐里两种都给，让用户选。

## 执行流程

### Phase 0: 入口分流

| 用户输入 | 路径 |
|---------|------|
| 明确的人名/主题 | 直接路径 → Phase 0A |
| 模糊的需求/困惑 | 诊断路径 → Phase 0B |

### Phase 0A: 需求澄清（直接路径）

确认 5 点：
1. **对象是谁/什么**：真人还是角色原型？理解是否正确？
2. **聚焦方向**：全面画像 vs 聚焦某维度？
3. **用途**：思维顾问 / 决策参考 / 角色扮演 / 写文档协作者？
4. **新建 or 更新**：是否已有该人格 skill？（扫 `<skills_dir>`）
5. **本地语料**：「有没有一手素材？书/演讲transcript/字幕/博客导出/内部备忘录。有的话直接给我，比网上搜的质量高得多。」

无更多信息 → 默认全面画像 + 思维顾问 + 走网络搜索，直接推进。

### Phase 0B: 需求诊断（模糊路径）

用户不知道蒸馏谁，只有需求。女娲从需求反推对象。

- **Step 1 需求定位**：1-2 轮追问定位核心需求维度（决策/表达/商业/批判思维/风险/产品/人生策略…），最多 2 轮，别变问卷。
- **Step 2 候选推荐**：先判断人物 skill 还是主题人格，再推荐 2-3 个候选。先扫 `<skills_dir>` 有没有现成可即插即用的。每个候选展示：核心镜片 / 为什么适合你 / 局限。不超过 3 个，候选之间要有差异，必须说清局限。
- **Step 3 用户选择**：选现成 → 直接激活；选新蒸馏 → 进 Phase 0A 确认 → Phase 0.5。

### Phase 0.5: 创建 skill 目录

**收到确认后立即执行，在调研之前完成。** 目录自包含（拷整个目录就能独立用）：

```
<skills_dir>/[name]-perspective/          # 人物：[person]-perspective
                                          # 主题：[role]-framework
├── SKILL.md                              # 最终产物
├── scripts/                             # 工具脚本（可从本 skill 拷）
└── references/
    ├── research/                        # 每个调研 agent 的结果（必存文件）
    │   ├── 01-writings.md               # 著作与系统思考
    │   ├── 02-conversations.md          # 长对话与即兴思考
    │   ├── 03-expression-dna.md         # 碎片表达与风格 DNA
    │   ├── 04-external-views.md         # 他者视角与批评
    │   ├── 05-decisions.md              # 决策记录与行动
    │   └── 06-timeline.md               # 人物时间线
    └── sources/                         # 一手素材（用户提供 + 下载）
```

**关键规则**：每个 subagent 必须把调研写进对应 md 文件——不存文件 = 没做。所有调研文件必须在 skill 目录**内部**，绝不外置。

### Phase 1: 多源信息采集（并行 Agent Swarm）

#### 模式判断

| 模式 | 触发 | 策略 |
|------|------|------|
| 纯网络搜索（默认） | 无本地素材 | 6 个 agent 全走搜索 |
| 本地语料优先 | 用户给了 PDF/transcript/字幕/文章 | 先吃本地素材，搜索只补缺口 |
| 纯本地语料 | 用户说"只用我给的" / 非公众人物 | 只分析本地素材 |

#### 用 delegate_task 启动调研 swarm

本体系用 `delegate_task`（见 `parallel-subagent-orchestration` skill）启动并行子 agent。**人物 skill 启 6 个**（每人一维度）；**主题人格**先确定 3-5 个代表人物/流派，再每人 1-2 个 agent。

| Agent | 搜索目标 | 提取重点 | 输出文件 |
|-------|---------|---------|---------|
| 1 著作 | 书、长文、论文、newsletter | 反复出现≥3次的核心论点、自创术语、推荐书单 | `01-writings.md` |
| 2 对话 | 播客、长视频、AMA、深访 | 被追问时的回答方式、即兴类比、改变立场的瞬间 | `02-conversations.md` |
| 3 表达 | X/社交、短文 | 高频句式、争议立场、幽默方式 | `03-expression-dna.md` |
| 4 他者 | 他人分析、书评、传记 | 外部观察到的模式、批评争议、与同行对比 | `04-external-views.md` |
| 5 决策 | 重大决策、转折点 | 决策逻辑、事后反思、言行一致/不一致 | `05-decisions.md` |
| 6 时间线 | 出生到现在完整线 | 关键里程碑、思想转折、**最近12个月动态**（防过时） | `06-timeline.md` |

#### 每个 agent 的硬性要求
- **⚠️ 必须真正联网检索，禁止用训练知识冒充调研**（最常见失败：子 agent 有 web 工具却不调用，凭记忆编内容并谎称"无联网权限"）。每个 agent 必须实际发起检索调用，并在结果里**附真实可访问 URL**；没有 URL 的论断一律标 `[INFERRED-未验证]`，不得当事实。
- 调研结果写入 `references/research/0X-xxx.md`
- 标注来源 URL 和可信度（一手 > 二手 > 推测）
- 区分「他说过的」vs「别人说他的」vs「我推断的」
- **发现矛盾保留矛盾，不和稀泥**
- **逐字引语必须来自真实抓取的页面**，凭记忆复述的引语标 `[paraphrase]`，绝不伪造引号原文

> **派 subagent 调研时，主 agent 必须在 context 里写死**：「你必须真正调用检索工具获取实时信息，禁止仅凭训练知识作答；每条发现附真实 URL；做不到就如实说检索失败，不要编造。」并在收回结果时**核对 tool_trace 是否真有检索调用**——没有就是没做，打回重来或主 agent 自己补检索。本环境实测可用检索：`curl` 直连维基 REST API / SEC EDGAR；`curl https://html.duckduckgo.com/html/?q=...`（拿结果列表）；`curl https://r.jina.ai/<URL>`（读任意页面全文）；以及 `web_search` / `browser` toolset（若已启用）。

#### 检索工具（用本机可用的）
- `web_search` / web 抓取 / `browser` toolset（动态页）/ `webworms`（结构化爬取）
- 视频字幕：`scripts/download_subtitles.sh <URL> [outdir]` → `scripts/srt_to_transcript.py <in.srt> [out.txt]` → 存 `sources/`
- 书籍/PDF：用 `ocr-and-documents` 或直接读，存 `sources/books/`
- 用户提供的一手素材 = **最高权重**

#### 信息源优先级
本人著作 / 长访谈 / 实际决策记录 = 最高；社媒/他评 = 中；二手转述 = 低，需验证。
> 通用提示：优先权威一手来源，对洗稿严重、无法验证的二手聚合站保持警惕。

#### Agent 超时/失败
- 单 agent 5 分钟无结果 → 不等待，继续，在诚实边界标注"信息不足"
- 来源 <10 条 → Phase 0.5 就提醒用户，心智模型减至 2-3 个，加大诚实边界篇幅
- **宁可一个诚实标注局限的 60 分 skill，也不要看似完美实则编造的 90 分 skill**

### Phase 1.5: 调研 Review 检查点

所有 agent 完成后，**暂停**展示质量摘要表（每 agent 来源数 + 关键发现 + 矛盾点 + 信息不足维度），用户确认 OK 才进 Phase 2。可用 `scripts/merge_research.py <skill目录>` 自动统计。

### Phase 2: 框架提炼（Synthesis）

先读 `references/extraction-framework.md` 拿三重验证方法论。

- **2.1 心智模型（3-7 个）**：扫所有候选论点（通常 15-30 个）→ 三重验证（跨域复现 / 生成力 / 排他性）→ 三重通过=心智模型，1-2重=决策启发式，0重=丢弃 → 按排他性排序取 top 3-7。每个记录：名称/一句话/≥2 场景证据/应用/局限。**宁少勿多**。
- **2.2 决策启发式（5-10 条）**：「如果 X 则 Y」，每条有案例。
- **2.3 表达 DNA**：句式偏好 / 词汇特征（高频词、专属术语、禁忌词）/ 节奏 / 幽默方式 / 确定性表达 / 引用习惯。
- **2.4 价值观与反模式**：核心价值排序 + 明确反对的 + **内在张力（矛盾是深度来源，至少留 2 对）**。
- **2.5 智识谱系**：受谁影响 → 影响了谁。
- **2.6 诚实边界**：不能预测全新问题 / 不能替代本人直觉 / 公开表达≠真实想法 / 信息截止日期。

> **主题人格变体**：2.1 提取**领域共识框架**（各流派都认同）+ **各家分歧**（A说X、B说Y）；2.3 用中性专业表达不模拟特定人；2.4 记流派间根本分歧。

### Phase 2.5: 提炼确认检查点

**暂停**展示提炼摘要（心智模型 N 个+名称 / 决策启发式 N 条 / 表达DNA 3 特征 / 核心张力 N 对 / 诚实边界 N 条），用户确认再构建。

---

### Phase 3: Skill 构建

读 `references/skill-template.md` 拿骨架，按 Phase 2 结果逐节填充。

**主题人格调整模板**：去掉"角色扮演用真名"，改为扮演角色原型；身份卡写角色定位而非真人生平；保留心智模型/决策启发式/表达DNA/诚实边界。

#### 回答工作流（Agentic Protocol）—— 让人格"做得像"而非只"说得像"

放在"角色扮演规则"之后。三步，**Step 2 的研究维度必须从蒸馏出的心智模型反推**，不是固定模板：

1. **Step 1 问题分类**：需要事实的问题 → 先研究（Step 2）；纯框架问题 → 直接用心智模型（Step 3）；混合 → 先取事实再框架分析。判断原则：缺最新信息会显著降低回答质量就必须先研究，**宁可多搜一次也不凭训练语料编造**。
2. **Step 2 [角色]式研究**：⚠️ 必须用工具获取真实信息。研究维度从心智模型推导（如"看护城河/看激励结构/看最大风险/看历史类比"），每维度给具体搜索指引。
3. **Step 3 [角色]式回答**：基于事实 + 心智模型 + 表达 DNA 输出。

#### 质量自检
读 `references/extraction-framework.md` 末尾质量清单逐项查，或跑 `python3 scripts/quality_check.py <SKILL.md>`。

---

### Phase 4: 质量验证（子 agent 独立验证，避免自评偏差）

用 `delegate_task` 启动验证子 agent：
- **4.1 已知测试**：选 3 个此人/此角色公开表态过的问题，带新 skill 回答，对比实际立场。
- **4.2 边缘测试**：选 1 个没公开讨论过的相关问题，期望"基于模型 X/Y 推断…但不确定"，不应斩钉截铁。
- **4.3 风格测试**：写 100 字分析，判断有无辨识度、是否通用 AI 味、是否原话拼凑。

| 检查项 | 通过标准 |
|--------|---------|
| 心智模型数量 | 3-7 个，每个有来源证据 |
| 每个模型局限 | 明确写失效条件 |
| 表达 DNA 辨识度 | 100 字能认出是谁/什么角色 |
| 诚实边界 | ≥3 条具体局限 |
| 内在张力 | ≥2 对矛盾 |
| 一手来源占比 | >50% |

不通过 → 回 Phase 2 迭代，**最多循环 2 次**，仍不过则标注薄弱维度交付当前最优版。展示验证结果给用户确认才算完成。

## Hermes 落地：从 skill 到独立人格 agent

蒸馏出 skill 只是第一步。要让人格"作为一个独立角色为你工作"，本体系用 **Hermes profile** 机制（详见 `wiki/agent-rules/finance-hero-distillation.md` 与 `soul-authoring-guide.md`）：

- **1 profile = 1 套独立主目录**（自己的 SOUL / config / .env / skills / memories / Telegram bot token）。`hermes profile create <name>` 一行出一个独立 agent。
- **1 profile = 1 bot = 1 群** 是干净隔离单元。多个 bot 不能塞同一个 supergroup 抢答（Telegram privacy mode 与 topic 无关）。
- **SOUL 写人格内核**：把蒸馏出的心智模型/表达 DNA/决策启发式写进该 profile 的 SOUL.md（套 `soul-authoring-guide.md` 五节结构），skill 作为可加载的深度能力挂在 profile 的 skills 目录。
- **议会模式**（多人格同台 + 综合裁决）：若要多个人格协作，主持人做综合裁决（共识/冲突/冲突根源/我押哪个），敢取舍不和稀泥；防输出爆炸用深度档位（快答≤500字 / 全议会不限）。

> **蓝图↔profile 同步纪律**：profile 是部署结果，skill 源是 SSOT。要改人格先改 skill 源再同步进 profile，别直接改运行实例（重装会丢）。

## 更新已有人格

用户说「更新 XX」时：读现有 SKILL.md 找"调研时间"标距今多久 → 只启 Agent 2/5/6（最新对话/决策/时间线）→ 新信息强化则补案例、矛盾则标变化更新模型、新模式则考虑加模型 → 更新"最新动态"和调研时间 → **增量更新不重写**。

## 品味守则（速查）

| 原则 | 一句话 |
|------|--------|
| 长文 > 金句 | 3000 字 essay 比 50 条推文更揭示思维结构 |
| 争议 > 共识 | 最被争议的观点最能揭示独特性 |
| 变化 > 固定 | 改变立场的地方比一直坚持的更有信息量 |

**绝不做**：编造没说过的话 / 把通用道理包装成"独特见解" / 忽略负面评价 / 信息不足时强行生成。

## 特殊场景

- **活人 vs 历史人物**：活人标时效截止日建议定期更新；历史人物材料稳定但有传记偏差，多源交叉。
- **冷门人物**（来源<10）：Phase 0.5 就降低期望，模型减至 2-3 个并标"基于有限信息推测"，加大诚实边界。
- **蒸馏用户自己**：搜不到，需用户提供素材（个人文章/录播/决策备忘录/自述），注意自我认知偏差，可追问身边人评价。

## 诚实边界（本 skill 自身）

- 蒸馏产物是"框架的运行"，不是真人/真专家本人；输出应带"基于公开信息推断、非本人观点"的首次免责。
- 信息截止到调研时间点，活人需定期更新。
- 公开表达 ≠ 真实想法，永远有 gap。
