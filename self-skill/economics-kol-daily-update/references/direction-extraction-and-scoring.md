# 结构化多空方向抽取 + 共识度评分（Chao 2026-06-21 两次重批）

> 触发场景：dashboard 多空指标/雷达图显示"所有资产大家都看多"、美债全看多、某板块全是"中性"、几乎所有板块评分都顶到 ±99/±100。这是两个独立设计缺陷，本文给完整修复方案。Chao 的最高质量红线。

---

## A. 方向抽取：绝不浅层匹配（第一次重批）

### 症状
dashboard 显示所有资产都看多——美债没人看空、Equities 没人看空、AI&Tech 板块全是"中性"。用户一眼识破不合理。

### 根因（三个独立 bug 叠加）
1. **没有结构化方向字段**——多空只藏在 Comments 自由文本 + 多空标的 emoji 里。
2. **聚合代码浅层文本/emoji 匹配**——`if "🟢" in text: bull`，没匹配到就 `return 0`(中性)。结果 38% 无 emoji 的记录全被算成中性，看空信号被静默吞掉。
3. **整条简化成单一方向**——一条发言常含多标的且方向相反，给整条打一个标签(按主基调)会丢掉其余腿(尤其看空腿)。

### 诊断（先证明是算法吞了，不是数据没有）
不要急着改算法。先统计原始数据 emoji 分布：本项目 🟢1239/🔴976 → 看空其实和看多几乎一样多 → 算法吞了。再按板块看 has🔴 比例：美债84条48条有看空、Equities129条63条有看空 → 看空一直在。

### 正解（三层）
1. **建结构化字段**（By Day DB）：
   - `方向明细`(rich_text 存 JSON 数组,**按标的拆分**)：`[{"标的":"美债","板块":"Government Debt","方向":"看空"},{"标的":"黄金","板块":"Precious Metals","方向":"强烈看多"}]`
   - `主导方向`(select 6档)：强烈看多/看多/中性/看空/强烈看空/分歧
   - Notion API 可直接把 rich_text 字段【改类型】成 select（PATCH database properties，自动迁移精确匹配的值，变体值变空需备份回填）。
2. **LLM 逐条读懂语言意味**（绝不浅层匹配）：
   - 识别隐含看空(无🔴也算)：「美债泡沫/收益率飙升/抛长债/缩久期/避开TLT」=看空美债；「科技泡沫/Mag7见顶/AI估值过高/纸面资产崩盘」=看空Equities；「美元购买力崩溃/去美元化」=看空美元；「黄金成拥挤交易/接近极端看涨」=(反向)看空黄金。
   - **按标的拆分,不整条简化**：Luke Gromen 一条=🟢黄金(强烈)+🟢BTC+🔴长久期美债(强烈看空)+🔴美元。
   - 中性=真无方向(非偷懒默认)；不无中生有(没提的不编)；占星/预言类(Alternative)也判方向。
3. **聚合读结构化字段**，摊平成 (板块,标的,方向) 三元组统计。一条发言因此给多个板块贡献不同方向。sector_summary 输出 legs_bull/legs_bear/strong_bear。未回填的旧记录回退文本法但标注。

### 工程模式（2000+ 条逐条 LLM 判断）
- 建处理库 `scripts/extract_direction.py`，命令：`count`(总数/已填/剩余) / `list <off> <lim>` / `next <lim>`(返回下一批未填记录) / 函数 `write_direction(page_id, detail_list, dominant, targets_str, std_sector)`(校验板块7标准+方向6档+rich_text分块+select合并)。
- **子 agent 每批 30 条**逐条读懂回填(>30 易超时)，并发上限 3，每批写后独立 `count` 验证。
- **余量用 cron 滚动**：每 8 分钟一个 agent 取 `next 50` 处理，remaining=0 时自停。
- **收尾 cron**：每 20 分钟检查，remaining>0 静默退出(回复"WAIT")，==0 才全量复查+重生 dashboard+主动汇报用户。
- ⚠️ 绝不用脚本/正则自动判方向——那就退回浅层匹配了，必须 LLM 逐条读。
- Sector 标准化映射表 `config/sector_standard_map.json`(35杂乱→7标准)，随方向回填一起改，避免对2000条 PATCH 两遍。

### 修复后验证（用数据自证）
美债 空76(强8) → 偏空 ✓；Macro 空155(强38) → 偏空 ✓；Equities 空70>多66 ✓。不再全板块正分。

## B. 评分公式：共识度净占比，不用 tanh（第二次重批）

### 症状
sector 评分几乎全部顶到 ±99/±100。贵金属有 113 条看空腿却显示 +99。用户："100分意义是没人有相反看法，有一个人相反就不该是100分。"

### 根因
`score = 100*tanh(加权和/sqrt(n))` 是"信号强度映射"——加权和一大 tanh 就饱和到 ±1，**完全不反映有多少人持相反看法**。

### 正解
```
score = 100 × (加权看多 − 加权看空) / (加权看多 + 加权看空 + 分歧)
```
强度加权：强烈看多=+2、看多=+1、看空=−1、强烈看空=−2；分歧计入分母不计净值。语义：
- **±100 只有零反对时才可能达到**(有一个相反立场就 <100)
- 多空各半 = 0
- score 本身 = "净多空倾向%"；`consensus = |score|` = 共识度
- 前端标签直接用 score(>20偏多/<−20偏空/中间分歧)，进度条宽 `|score|/2`

### 修复后验证
贵金属 +90(有113空,不再+99)；美债 −96(几乎全场看空,合理接近极值)；**Equities 旧+57误判→新−20(诚实反映空291>多184)**。±100 成为"全场一致"极值而非默认饱和。

## 通用红线（已升级到 wiki/llm/sentiment-direction-extraction.md）
情绪/方向分析绝不可浅层文本/emoji 匹配或默认中性。必须逐条读懂语言意味 + 按标的拆分 + 存结构化方向字段 + 聚合读字段。评分公式必须让"分歧度进公式"——有反对就不该满分。
