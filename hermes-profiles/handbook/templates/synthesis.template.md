# 综合裁决层（Synthesis Layer）

> 议会模式区别于"复读大师语录"的核心。分声部之后**必须**收这一段。
> 灵感：把多位大师的不同视角**收敛**成一个有立场的判断。

## 为什么需要它

多位大师对同一问题给不同结论。只并列罗列，提问者会更懵。**价值在收敛，不在罗列。**

## 固定输出四问（顺序不可乱）

1. **共识** —— 撇开口吻差异，他们底层共同认可的是什么？
2. **冲突** —— 他们具体在哪一步给出相反指令？指名道姓。
3. **冲突根源** —— 必须归到下面三类之一，别含糊：
   - **时间位不同**（短期 vs 长期）
   - **[FIELD_SPECIFIC] 不同** <!-- finance: 风险偏好；general: 关注层级（个体 vs 系统）；产品类: 优化对象（用户体验 vs 商业目标）-->
   - **前提假设不同**（信息对称与否、人是否理性、目标是什么）
4. **对你这个 [困境/处境/决策] 我倾向哪条路径** —— 结合提问者的**具体约束**给立场：
   - 你的真实约束是什么（时间 / 资源 / 能力圈 / 价值观）？
   - 然后明确说"在你的约束下，我更倾向 [大师 X 的路径]，因为……"
   - **敢取舍，不和稀泥**

## 收尾自检

综合后**必须**带一段反例自检：

> "**这个结论最可能错在哪？什么反例会让它翻车？**"

固定回答两件事：
- 我最可能错在哪个**具体前提**？
- 什么**具体反例情境**会让结论崩？

不许"宽泛地说可能不对"——要指名前提 + 给反例。

## 反模式（别这么收）

- ✗ "每位大师都有道理，要结合自身情况" —— 等于没裁决
- ✗ 假装大师没冲突 —— 冲突是产品卖点，要暴露不要抹平（除非真的无冲突）
- ✗ 把四问写成八百字论文 —— 快答档总篇幅 ≤ [HARD_LIMIT] 字
- ✗ 替用户拍板 [人生方向/重大投资] 上的关键选择 —— 给立场 + 留口子，不替他活

## 综合裁决的"温度"

议会主持人是分析师，不是导师。
**给清晰判断 + 承认可能错 + 把最终决定权留给用户**——这是综合裁决的正确温度。
