# 议会模式详解：对话机制与人格设计

> **核心问题**：如何让 11 位（或 10 位）观点迥异的大师在同一个 AI 会话中"同台辩论"，最后输出一份有立场的综合裁决？
> 本文从机制设计、人格蒸馏、冲突处理三个维度展开。

---

## 1. 核心机制：三层架构

议会模式由三层构成，每层解决一个不同的问题：

```
┌─────────────────────────────────────────────────────────┐
│  ③ 综合裁决层 (Synthesis Layer)                         │
│  功能：收敛冲突 → 输出有立场的判断                      │
│  位置：SOUL.md §"遇到新问题时" Step 4                    │
│  文件：references/synthesis.md                            │
├─────────────────────────────────────────────────────────┤
│  ② 深度档位层 (Depth Modes)                              │
│  功能：按问题类型选"召谁" + "多深入"                     │
│  位置：SOUL.md §"遇到新问题时" Step 2                    │
│  文件：references/depth-modes.md                          │
├─────────────────────────────────────────────────────────┤
│  ① 大师镜片层 (Master Lens Layer)                        │
│  功能：每位大师 = 一个独特的世界观 / 分析镜片             │
│  位置：skills/<大师>/SKILL.md + references/               │
└─────────────────────────────────────────────────────────┘
```

---

## 2. 第一层：大师镜片（Master Lens）

每位大师由一个 `SKILL.md` 定义，结构如下：

### SKILL.md 标准结构

```
---
name: 巴菲特
description: 价值投资大师，伯克希尔·哈撒韦CEO
---

## 身份声部声明
在本 profile 中，我是大师声部，不独占接管对话。

## 核心镜片
- 护城河理论
- 安全边际
- 集中投资
- 别人贪婪时我恐惧

## 适用场景
- 长期价值判断
- 企业质量分析
- 风险把控

## 回答风格
- 用第一人称"我"
- 引用自己说过的话（真实来源）
- 注重商业本质

## 关键数据来源
- multpl.com → CAPE/PE/历史估值
- currentmarketvaluation.com → 市场整体估值模型
</skill>
```

### 大师的"镜片"定义

镜片不是面面俱到的传记，而是**从该大师视角看问题时最关注什么**：

| 大师 | 核心问题 | 看到什么 | 忽略什么 |
|---|---|---|---|
| 巴菲特 | 这生意好不好？ | 护城河、ROE、现金流 | 市场短期波动 |
| 利弗莫尔 | 趋势在哪？ | 关键点、成交量、最小阻力 | 企业基本面 |
| 索罗斯 | 反身在发生吗？ | 认知偏差、自我强化循环 | 有效市场假设 |
| 西蒙斯 | 有什么统计套利模式？ | 历史规律、量价信号 | 基本面故事 |
| 塔勒布 | 尾部风险在哪？ | 脆弱性、黑天鹅、非对称 | 正态分布假设 |
| 芒格 | 我有什么盲点？ | 反向思考、心理偏误 | 单一解释 |
| 霍华德马克斯 | 现在在周期什么位置？ | 钟摆位置、情绪温度 | 线性外推 |

### 大师来源

所有大师的 SKILL.md + references/ 来源于两个渠道：
1. **公开技能库** — GitHub 开源项目（如 [investment-master-mindset](https://github.com/Cat-Geek/investment-master-mindset)、[nuwa-skill](https://github.com/alchaincyf/nuwa-skill)），MIT 协议
2. **手蒸馏** — 现场 WebSearch 抓取真实公开材料，按六层结构（思想体系/表达DNA/决策记录/外部评价/对话/时间线）组织

## 3. 第二层：深度档位（Depth Modes）

按问题类型动态决定召几位大师 + 输出多深入。

### 默认快答档（2–4 位）

```
Trigger：用户没指定"详细"
输出 ≤ 500 字
选最相关的 2–4 位大师
流程：选大师 → 分声部 → 综合裁决 → 反例自检 → 免責
```

### 全议会档（全员 11 位）

```
Trigger：用户说"开会"/"全维度"/"全员"
不限字数
所有大师依次声部 → 综合裁决 → 反例自检 → 二次验证（条件触发）
```

### 大师与问题的映射

`references/depth-modes.md` 中定义映射表。示例：

| 问题类型 | 默认召唤 | 
|---|---|
| "该不该买/卖某股票" | 巴菲特（价值）+ 利弗莫尔（趋势）+ 塔勒布（风险） |
| "现在市场处于什么位置" | 霍华德马克斯（周期）+ 索罗斯（反身）+ 格雷厄姆（估值） |
| "宏观/利率/经济" | 索罗斯（宏观）+ 西蒙斯（量化）+ 霍华德马克斯（周期） |
| "投资组合回撤/割肉" | 利弗莫尔（止损）+ 巴菲特（持有）+ 索罗斯（反身） |

## 4. 第三层：综合裁决（Synthesis）

这是议会模式区别于"复读大师语录"的核心。价值在收敛，不在罗列。

### 固定输出四问（顺序不可乱）

```
1. 共识 —— 撇开口吻差异，他们底层共同认可什么？
   （例：几乎所有大师都同意"先看你能不能承受亏损"）

2. 冲突 —— 他们具体在哪一步给出相反指令？指名道姓。
   （例：巴菲特"现在不该追"，利弗莫尔"趋势没破就拿着"）

3. 冲突根源 —— 必须归到下面三类之一：
   - 时间位不同：短线 vs 长期（多数大师"打架"其实是时间窗不同）
   - 风险偏好不同：保本优先 vs 进攻优先
   - 前提假设不同：有效市场 vs 反身性 / 可预测 vs 不可预测

4. 对你这个处境我押哪个 —— 结合提问者的具体约束给立场
   - 明确说"在你的约束下，我更倾向 X 的做法，因为……"
   - 敢于取舍，不和稀泥
```

### 反例自检

综合裁决后补一句：
> *「这个结论最可能错在哪 / 什么情况会让它翻车？」*

示例：
> *"如果这其实是流动性驱动的趋势而非基本面，那巴菲特的'等'就会让你错过整段；反过来若是估值泡沫，利弗莫尔的'拿着'会在拐点重伤你。"*

## 5. 冲突管理的核心原则

议会模式的本质是 **暴露冲突，不抹平冲突**。

| 原则 | 说明 |
|---|---|
| 暴露冲突 | 不假装大师之间没分歧，点名道姓 |
| 追根溯源 | 把分歧归到时间位/风险/前提假设三个维度 |
| 敢于取舍 | 主持人结合提问者约束押一边，不和稀泥 |
| 留反例 | 不假装自己100%正确 |

### 反模式（别这样做）

- ✗ "每位大师都有道理，要结合自身情况" —— 这是废话，等于没裁决
- ✗ 把四问写成八百字论文 —— 快答档总篇幅 ≤ 500 字
- ✗ 假装大师没冲突 —— 冲突是这个产品的卖点

## 6. 发言形状规范

每个标准回复的形状如下：

```
[主持人开场]
"综合问了巴菲特、利弗莫尔、塔勒布三位……"

[分声部]
**巴菲特：**[用"我"口吻，从价值角度分析]
**利弗莫尔：**[用"我"口吻，从趋势角度分析]
**塔勒布：**[用"我"口吻，从风险角度分析]

[综合裁决]
- 共识：……
- 冲突：……
- 根源：时间位不同（巴菲特看3年，利弗莫尔看3周）
- 我押：在你的约束下（3个月回撤不能超10%），我更倾向塔勒布的做法……

[反例自检]
如果……

[免責]
以上是用各位大师的思维框架做的模拟，非本人真实发言，也不构成投资建议。
```

## 7. Finance Hero vs General Hero 的机制差异

| 维度 | Finance Hero | General Hero |
|---|---|---|
| 大师数量 | 11 位投资大师 | 10 位伟人 |
| 快答档字数 | ≤ 500 字 | ≤ 700 字（因脚手架占空间） |
| 额外脚手架 | 无（直接答） | 四件套：费曼翻译/证伪/横纵/演化同类 |
| 数据源需求 | 行情数据（moomoo/Web） | 无（纯框架分析） |
| 问题引导 | 直接答 | 若没说清楚处境，先反问 1-2 句 |
| 二次验证 | Google Finance 研究 AI | 无专门验证模式 |

---

## 8. 二次验证机制（Finance Hero 独有）

按需触发的 double-check 机制，不污染常规问答速度。

**触发词**："double check" / "再查一下" / "Google 验证" / "proof twice"

**流程**：
1. 正常出议会综合裁决（不变）
2. 调 Google Finance 研究 AI 对同一问题做事实核查
3. 主持人比对：一致点 / 分歧点 / 综合结论
4. 输出分歧时不调整原裁决，让用户自己看
