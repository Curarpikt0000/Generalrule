# LLM 情绪/多空方向抽取：绝不可浅层文本匹配

> 来源：Economy-KOL-to-Notion 项目，Chao 2026-06-21 重批。这是 LLM 做任何"情绪/立场/方向"结构化抽取的通用红线。

## 症状（怎么发现踩坑的）

dashboard 的多空情绪雷达图显示**"所有资产大家都看多"**——美债全看多、股票板块没一个看空、某板块 20 条全是"中性"。用户一眼识破：不可能所有资产都看多，尤其美债怎么可能没人看空。

## 根因（三个独立 bug 叠加）

1. **没有结构化方向字段**：方向只藏在自由文本（comments）+ emoji（🟢🔴）里，没有专门的结构化字段存"判断结果"。
2. **浅层文本/emoji 匹配**：聚合代码用 `if "🟢" in text: bull` 这种规则，**没匹配到就 return 中性**。结果：大量没打 emoji 的记录（本项目 38%）被全部算成中性，看空信号被静默吞掉。
3. **整条简化成单一方向**：一条发言常包含多个标的且方向相反（同一个 KOL：看多金银 🟢 + 看空美债 🔴 + 看空科技股 🔴）。若给整条打一个方向标签（按主基调），其余标的的方向（尤其看空腿）全部丢失。

## 诊断方法（先证明是算法吞了，不是数据没有）

不要急着改算法。先**统计原始数据的方向信号分布**：
- 全库 emoji 计数：🟢1239 / 🔴976 → 看空其实和看多几乎一样多 → 证明数据里有看空，是聚合算法吞了。
- 按板块看"含看空信号"比例：美债 84 条里 48 条含看空、股票 129 条里 63 条含看空 → 看空信号一直在。

## 正解（三层修复）

### 1. 建结构化方向字段
- `方向明细`（存 JSON 数组，**按标的拆分**）：`[{"标的":"美债","板块":"Government Debt","方向":"看空"}, {"标的":"黄金","板块":"Precious Metals","方向":"强烈看多"}]`
- `主导方向`（select，多档强度）：强烈看多/看多/中性/看空/强烈看空/分歧

### 2. LLM 逐条读懂语言意味（绝不浅层匹配）
- **识别隐含看空**（无显式看空词/emoji 也要判出）：
  - "美债泡沫 / 收益率飙升 / 抛售长债 / 缩短久期 / 避开 TLT" = 看空美债
  - "科技股泡沫 / Mag7 见顶 / AI 估值过高 / 纸面资产崩盘" = 看空股票
  - "美元购买力崩溃 / 去美元化 / 法币贬值" = 看空美元
  - "黄金成为拥挤交易 / 接近极端看涨区域" = （反向）看空黄金
- **按标的拆分，不整条简化**：一条发言对每个提到的标的分别判方向。
- **不滥用中性**：中性 = 真的只陈述事实无方向，不是"懒得判就给中性"。
- **不无中生有**：没提到的标的不编。

### 3. 聚合层读结构化字段，摊平成三元组
- 把所有记录的 `方向明细` 摊平成 `(板块, 标的, 方向)` 三元组统计。
- 一条发言因此能给**多个板块**贡献**不同方向**。
- 板块多空 = 该板块所有腿的方向分布（legs_bull/legs_bear/strong_bear），不是按"行/人头"。
- 标签判定综合"加权分数 + 腿级真实偏向"，避免时间衰减后分数偏中性但实际压倒性看空。

## 修复后验证（用数据自证）

- 美债：空 76（强烈看空 8）→ 偏空 ✓（修复前显示全看多）
- 宏观/美元：空 155（强烈看空 38）→ 偏空 ✓
- 股票：空 70 > 多 66 ✓（修复前显示没人看空）
- 不再是所有板块都正分——贵金属/能源偏多，美债/宏观偏空，这才是合理的市场全景。

## 工程模式（大规模逐条判断）

2018 条规模：建一个处理库（list/next/count/write 命令）→ 子 agent 每批 30 条逐条读懂回填 → 每批写后独立 count 验证 → 余量用 cron 滚动后台跑完。不要用脚本自动判方向（那就退回浅层匹配了），必须 LLM 逐条读。

## 一句话红线

**情绪/方向分析绝不可用浅层文本/emoji 匹配或默认中性。必须逐条读懂语言意味 + 按标的拆分 + 存结构化方向字段 + 聚合层读字段而非现猜文本。**
