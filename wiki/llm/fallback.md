# LLM 调用的级联回退（Fallback）设计准则

本文档定义了在应用中调用大语言模型（LLM）时，为确保服务鲁棒性和高可用性而必须遵守的级联回退策略。

## 核心原则
放弃静态、硬编码的模型调用方式，建立一个动态、多层次、具备自动容错能力的调度系统。

## 实现步骤
1. **禁止硬编码模型 ID**：所有标识符必须从配置或动态发现中获取。
2. **动态发现可用模型**：实时查询当前 API 权限下可用的模型列表并缓存。
3. **级联回退 (Cascading Fallback)**：
   - 第一优先级：Pro 级模型。
   - 第二优先级：Flash 级模型（自动降级）。
   - 兜底方案：跨平台切换（如从 Vertex AI 切换至 AI Studio）。
4. **完整容错回馈**：异常信息必须包含完整的失败诊断路径。
